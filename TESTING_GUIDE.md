# 企业微信客服与支付功能测试指南

## 问题1：企业微信客服功能是否需要用户微信登录？

### 答案：不需要

企业微信客服功能**不需要用户用微信登录**就可以正常使用。

### 企业微信客服的三种实现方式

#### 方式1：企业微信客服API（推荐，无需登录）

**工作流程**：
1. 用户在前端点击客服按钮
2. 系统调用企业微信客服API，生成客服对话链接
3. 跳转到企业微信客服对话页面（在微信中打开）
4. 用户可以直接在微信中与客服沟通
5. 客服在微信中回复消息

**优点**：
- ✅ 用户无需登录
- ✅ 使用微信原生体验
- ✅ 消息直接到达微信
- ✅ 支持图片、文件等富媒体

**适用场景**：
- 售前咨询
- 售后服务
- 一般性咨询

#### 方式2：显示客服二维码（推荐，无需登录）

**工作流程**：
1. 用户在前端点击客服按钮
2. 显示客服二维码
3. 用户扫码添加企业微信客服
4. 用户可以在微信中与客服沟通

**优点**：
- ✅ 用户无需登录
- ✅ 用户可以添加客服为好友
- ✅ 后续可直接联系
- ✅ 适合长期客户关系

**适用场景**：
- VIP客户服务
- 长期合作客户
- 需要持续沟通的场景

#### 方式3：内置聊天窗口（需要登录）

**工作流程**：
1. 用户在前端点击客服按钮
2. 显示聊天窗口
3. 用户输入消息
4. 系统将消息转发到企业微信客服
5. 客服在微信中回复
6. 系统将回复显示在前端

**优点**：
- ✅ 用户停留在网站
- ✅ 可以记录用户身份
- ✅ 可以关联订单信息

**缺点**：
- ❌ 需要用户登录
- ❌ 需要后端转发消息
- ❌ 实现较复杂

**适用场景**：
- 已登录用户
- 需要关联订单的咨询
- 需要记录历史的沟通

### 当前项目实现情况

当前项目实现了**方式2（客服二维码）**和**方式3（内置聊天窗口）**的简化版本：

1. **客服二维码模式**：用户点击客服按钮，显示客服二维码，扫码添加
2. **聊天窗口模式**：用户点击客服按钮，显示聊天窗口，显示欢迎消息和常见问题

这两种方式**都不需要用户登录**即可使用。

### 如果用户登录了会有什么不同？

如果用户登录了，系统可以：
- 在聊天窗口显示用户昵称
- 关联用户的订单信息
- 记录用户的历史咨询
- 提供个性化的服务

但**不是必需的**，未登录用户也可以正常使用客服功能。

## 问题2：如何在测试环境中测试支付功能和企业微信功能？

### 答案：需要部署到公网环境

#### 为什么需要公网环境？

**微信支付和微信客服功能都需要**：
1. 微信服务器能够访问你的服务器（回调URL）
2. 配置的域名必须是公网可访问的
3. 不能使用 `localhost` 或内网IP

**原因**：
- 微信支付需要回调到你的服务器通知支付结果
- 企业微信需要回调到你的服务器接收消息
- 这些回调请求来自微信服务器，必须能访问你的服务器

### 测试环境部署方案

#### 方案1：使用 ngrok（推荐，快速测试）

**ngrok** 是一个内网穿透工具，可以将本地开发环境映射到公网域名。

**步骤**：

1. **安装 ngrok**
   ```bash
   # macOS
   brew install ngrok

   # Windows
   # 从 https://ngrok.com 下载
   ```

2. **启动 ngrok**
   ```bash
   ngrok http 5000
   ```

3. **获取公网域名**
   ```
   Forwarding  https://abc123.ngrok.io -> http://localhost:5000
   ```

4. **配置微信支付**
   - 支付授权目录：`https://abc123.ngrok.io/`
   - 回调域名：`abc123.ngrok.io`
   - 回调地址：`https://abc123.ngrok.io/api/payment/wechat/jsapi/callback`

5. **配置企业微信**
   - 回调URL：`https://abc123.ngrok.io/api/wecom/callback`
   - 使用 ngrok 域名配置企业微信

6. **测试支付功能**
   - 在 ngrok 环境中创建订单
   - 发起支付
   - 在微信中完成支付
   - 检查订单状态

**优点**：
- ✅ 快速，无需购买域名
- ✅ 免费（有限制）
- ✅ 适合开发测试

**缺点**：
- ❌ 域名会变化
- ❌ 免费版有限制
- ❌ 不适合生产环境

#### 方案2：使用临时测试域名（推荐）

**临时测试域名**服务：
- **natapp.cn**
- **frp**
- **serveo.net**

**使用示例**：

```bash
# 使用 natapp
natapp -authtoken=your_token

# 使用 serveo
ssh -R 80:localhost:5000 serveo.net
```

**优点**：
- ✅ 免费
- ✅ 稳定
- ✅ 适合测试

#### 方案3：购买测试域名（推荐用于正式测试）

**步骤**：

1. **购买域名**
   - 阿里云、腾讯云等
   - 选择便宜的域名（如 .top, .xyz）
   - 费用约 10-50 元/年

2. **配置 DNS**
   ```
   # 将域名解析到你的服务器IP
   A    test    服务器IP
   ```

3. **配置服务器**
   - 安装 Nginx
   - 配置 SSL 证书（Let's Encrypt 免费）
   - 配置反向代理到本地 5000 端口

4. **配置微信**
   - 支付授权目录：`https://test.yourdomain.com/`
   - 回调域名：`test.yourdomain.com`

**优点**：
- ✅ 域名固定
- ✅ 可以长期使用
- ✅ 可以配置 HTTPS
- ✅ 适合正式测试

**缺点**：
- ❌ 需要购买域名
- ❌ 需要配置服务器

### 微信支付测试方案

#### 方案1：使用微信支付沙箱环境（推荐）

微信支付提供了**沙箱环境**，可以使用虚拟资金测试。

**步骤**：

1. **登录微信支付商户平台**
   - https://pay.weixin.qq.com

2. **进入沙箱环境**
   - 产品中心 → 沙箱

3. **开启沙箱环境**
   - 获取沙箱商户号
   - 获取沙箱 API 密钥

4. **配置项目**
   - 使用沙箱商户号
   - 使用沙箱 API 密钥
   - 其他配置相同

5. **测试支付**
   - 创建订单
   - 发起支付
   - 使用虚拟资金支付
   - 测试退款功能

**优点**：
- ✅ 无需真实资金
- ✅ 可以反复测试
- ✅ 不会产生真实交易
- ✅ 官方推荐方案

**缺点**：
- ❌ 沙箱环境功能有限
- ❌ 与真实环境有差异

#### 方案2：使用真实环境小额测试（不推荐）

**步骤**：

1. **创建小额订单**
   - 金额设置为 0.01 元
   - 测试基本流程

2. **支付后申请退款**
   - 在 365 天内可以申请退款
   - 退款不收取手续费

**优点**：
- ✅ 真实环境测试
- ✅ 可以测试完整流程

**缺点**：
- ❌ 需要真实资金
- ❌ 退款流程复杂
- ❌ 不适合频繁测试

## 常见问题

### Q1: 可以使用 localhost 测试吗？

**A**: 不可以。微信服务器无法访问 localhost。

### Q2: 可以使用 IP 地址测试吗？

**A**: 不可以。微信只接受域名配置。

### Q3: ngrok 域名会变化吗？

**A**: 免费版每次重启会变化，付费版可以固定。

### Q4: 可以在内网测试吗？

**A**: 可以测试前端显示，但无法测试微信支付和回调功能。

### Q5: 必须使用 HTTPS 吗？

**A**: 是的。微信只接受 HTTPS 配置（ngrok 自动提供 HTTPS）。
