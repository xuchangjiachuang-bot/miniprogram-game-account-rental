# 微信支付配置修复说明

## 问题描述

管理后台微信支付配置保存后，切换页面后配置重置，特别是敏感信息（如 API 密钥）会变成 `***`，保存后丢失。

## 问题原因

1. **后端返回隐藏值**：为了安全，后端在返回敏感信息时会隐藏（如 `api_key` 返回 `abc***def`）
2. **前端直接填充**：前端将隐藏的值直接填充到表单中
3. **保存时覆盖**：用户保存时，隐藏的值（`abc***def`）被保存到数据库，覆盖真实值

## 解决方案

### 1. 前端加载配置时
- 检测值是否包含 `***`
- 如果包含 `***`，则不更新表单字段，保持为空
- 这样用户看到敏感字段为空，需要重新填写

### 2. 前端保存配置时
- 过滤掉空值
- 过滤掉包含 `***` 的值
- 只保存用户实际填写的有效值

### 配置项详细说明

#### 1. 证书路径（cert_path）
- **含义**：商户证书文件的相对路径
- **格式**：`.p12` 格式
- **用途**：用于退款等功能的双向认证
- **示例**：`certs/wechat/apiclient_cert.p12`
- **获取方式**：
  1. 登录微信支付商户平台
  2. 进入"账户中心" → "API安全"
  3. 下载商户证书（apiclient_cert.p12）
  4. 将文件放到项目 `certs/wechat/` 目录下

#### 2. 密钥路径（key_path）
- **含义**：商户私钥文件的相对路径
- **格式**：`.pem` 格式
- **用途**：用于签名验证和退款
- **示例**：`certs/wechat/apiclient_key.pem`
- **获取方式**：
  1. 使用 `python3 extract_cert_full.py` 脚本从 `.p12` 文件提取
  2. 脚本会自动生成 `.pem` 文件
  3. 文件会保存在 `certs/wechat/` 目录下

#### 3. 异步通知地址（notify_url）
- **含义**：支付结果回调地址
- **用途**：微信支付会将支付结果通知到此地址
- **示例**：`https://hfb.yugioh.top/api/payment/wechat/jsapi/callback`
- **配置要求**：
  1. 必须是 HTTPS 地址
  2. 域名必须在微信支付商户平台配置
  3. 必须可公网访问
  4. 建议使用默认地址

#### 4. 证书密码（cert_p12_password）
- **含义**：.p12 证书文件的密码
- **格式**：通常就是商户号
- **用途**：解密 .p12 证书文件
- **示例**：`1106605743`
- **注意**：
  - 密码就是商户号
  - 不需要修改，使用默认值即可

#### 5. 证书序列号（cert_serial_no）
- **含义**：证书的唯一标识
- **格式**：40 位十六进制字符串
- **用途**：用于微信支付退款接口验证
- **示例**：`15FCE18FC26A4A9530CDADDC5FC2FDB10499F06B`
- **获取方式**：
  1. 使用 `python3 extract_cert_full.py` 脚本提取
  2. 脚本会自动显示证书序列号
  3. 查看 `CERTIFICATE_INFO.md` 文档

## 使用流程

### 首次配置

1. **获取基本信息**
   - 登录微信支付商户平台
   - 获取 AppID 和商户号
   - 设置 API 密钥

2. **下载证书**
   - 在"账户中心" → "API安全"下载商户证书
   - 将证书文件放到 `certs/wechat/` 目录
   - 运行 `python3 extract_cert_full.py` 提取证书信息

3. **配置回调域名**
   - 在微信支付商户平台配置回调域名：`hfb.yugioh.top`
   - 配置支付授权目录：`https://hfb.yugioh.top/`

4. **填写管理后台配置**
   - 访问管理后台 `/admin/payment/wechat`
   - 填写 AppID、商户号、API 密钥
   - 填写证书路径、密钥路径、证书密码、证书序列号
   - 点击"保存配置"

### 修改配置

1. 访问管理后台 `/admin/payment/wechat`
2. 修改需要更改的字段
3. 点击"保存配置"
4. 配置立即生效，无需重启服务器

### 重要提示

- **敏感信息安全**：
  - API 密钥、AppSecret 等敏感信息加载时会自动隐藏
  - 需要修改时重新填写即可
  - 不填写不会覆盖原值

- **证书管理**：
  - 证书文件不要提交到版本控制系统
  - 定期更新证书（有效期 5 年）
  - 证书密码通常就是商户号

- **回调地址**：
  - 确保回调地址可公网访问
  - 确保域名已在微信支付商户平台配置
  - 不要随意修改回调地址

## 常见问题

### Q1: 为什么敏感信息会显示 `***`？
A: 为了安全，后端在返回敏感信息时会隐藏。这不影响功能，需要修改时重新填写即可。

### Q2: 保存后配置会丢失吗？
A: 不会。敏感信息加载时为空，保存时不会更新这些字段，原值保留。

### Q3: 证书文件在哪里？
A: 证书文件应该放在 `certs/wechat/` 目录下，包括 `.p12` 和 `.pem` 文件。

### Q4: 如何获取证书序列号？
A: 运行 `python3 extract_cert_full.py` 脚本，会自动提取并显示证书序列号。

### Q5: 回调地址可以修改吗？
A: 可以修改，但需要同时在微信支付商户平台配置新的回调域名。

### Q6: 修改配置后需要重启服务器吗？
A: 不需要。配置存储在数据库中，修改后立即生效。
