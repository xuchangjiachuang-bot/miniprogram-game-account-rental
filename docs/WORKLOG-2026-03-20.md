## Security / Risk Tightening In This Round

The following routes were identified as inappropriate for open public access in a trial-operation environment and were tightened to admin-only access:

- `/api/init-db`
- `/api/debug/wechat-config`
- `/api/test/config-priority`
- `/api/test/wechat-check`
- `/api/test/wechat-qr-flow`
- `/api/test/wechat-urls`

Why this matters:

- these routes exposed configuration analysis, partial secret previews, or internal diagnosis output
- they are useful for maintenance, but should not stay publicly callable
- restricting them reduces accidental leakage and lowers the attack surface without changing normal user workflows

## Legacy / Old-Code Cleanup In This Round

The following legacy file was removed because it was not part of the active implementation and contained placeholder logic that could mislead future edits:

- `src/lib/wechat-pay-service.ts`

Reason for removal:

- no active imports
- mock / incomplete WeChat payment and transfer logic
- outdated environment variable assumptions
- easy to confuse with the real active `v3` payment path

Additionally, the withdrawal application route now avoids automatically copying platform real-name data into the withdrawal record account name field:

- `src/app/api/withdrawals/route.ts`

This keeps platform identity data decoupled from WeChat transfer parameters unless the transfer path explicitly requires additional fields.

## Remaining Known Risks

- top-level documentation is still fragmented across several historical reports and handoff notes
