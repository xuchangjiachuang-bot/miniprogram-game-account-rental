## 📖 概述

### 项目定位

**项目类型**：游戏账号租赁交易平台  
**核心游戏**：三角洲行动（Delta Force）  
**主要功能**：账号浏览、租赁下单、支付结算、聊天沟通、钱包管理

### 核心业务流程

```
用户登录 → 浏览账号 → 筛选账号 → 查看详情 → 下单支付 → 接收账号信息 
  ↓
开始租赁 → 验收账号 → 正常使用 → 归还账号 → 待卖家验收 → 分账结算 → 订单完成
  ↓
余额提现 → 评价订单
```

### 用户业务规则

#### 登录认证

```
Web端登录流程：
1. 输入手机号
2. 获取验证码（调用短信API）
3. 输入验证码
4. 后端验证码是否正确
5. 生成JWT Token
6. 返回Token和用户信息
7. 前端保存Token到LocalStorage

小程序端登录流程：
1. 用户点击授权登录
2. 调用wx.login()获取code
3. 后端调用微信code2Session获取openid和unionid
4. 通过unionid查找已有用户
   - 找到：直接登录，保存miniprogramOpenid
   - 找不到：提示绑定手机号
5. 用户绑定手机号（授权或手动输入）
6. 生成JWT Token
7. 返回Token和用户信息
8. 前端保存Token到Storage
```

#### 实名认证

```
实名认证流程：
1. 用户进入实名认证页面
2. 输入真实姓名（2-20个中文）
3. 输入身份证号（18位）
4. 点击提交
5. 后端验证身份证格式
6. 更新用户认证状态为pending
7. 管理员审核通过后，状态变为verified
8. 认证成功后，用户可以发布账号和提现

验证规则：
- 姓名：2-20个中文字符
- 身份证：18位，最后一位可以是X
- 每个用户只能认证一次
- 认证失败后可以重新认证
```

### 账号业务规则

#### 定价公式

```
租金（rentalPrice）：
- 基础价格：根据账号等级、段位、哈夫币数量计算
- 计算公式：
  basePrice = (coins * 0.001) + (accountLevel * 1) + (rankLevel * 5)
  rentalPrice = basePrice * (rentalDuration / 24)

押金（deposit）：
- 固定押金：50元（平台可配置）
- 高价值账号：押金 = max(50, coins * 0.01)

总价（totalPrice）：
- totalPrice = rentalPrice + deposit

示例：
- 哈夫币：10000
- 等级：Lv.10
- 段位：钻石（rankLevel=4）
- 租期：24小时

计算：
- basePrice = (10000 * 0.001) + (10 * 1) + (4 * 5) = 10 + 10 + 20 = 40
- rentalPrice = 40 * (24/24) = 40元
- deposit = 50元
- totalPrice = 40 + 50 = 90元
```

#### 发布账号流程

```
1. 用户进入发布账号页面
2. 填写账号信息：
   - 账号名称
   - 登录方式（微信/QQ/游客/其他）
   - 登录账号
   - 登录密码
   - 哈夫币数量
   - 比例
   - 安全箱状态
   - 体力等级
   - 负重等级
   - 账号等级
   - 段位
   - KD值
   - 地区
   - 皮肤（多选）
   - 上传截图（至少1张，最多10张）
   - 标签（可选）
   - 账号描述（可选）
   - 租期选择（24小时/48小时/72小时/自定义）
3. 系统自动计算租金、押金、总价
4. 用户确认信息
5. 提交发布
6. 后端验证信息完整性
7. 账号状态设为pending（待审核）
8. 管理员审核通过后，状态变为active（已上架）
```

#### 账号审核规则

```
审核标准：
✅ 账号信息完整
✅ 至少上传1张截图
✅ 哈夫币数量真实
✅ 登录信息可用
✅ 价格合理

审核操作：
- 通过：状态改为active（已上架）
- 拒绝：状态改为rejected（已拒绝），需填写拒绝理由
- 拒绝后可以重新提交
```

#### 账号上下架规则

```
上架：
- 状态为active的账号可以正常被浏览和下单
- 状态为rented的账号暂时不可见（已被租出）

下架：
- 卖家可以主动下架账号
- 管理员可以下架违规账号
- 下架后账号不可见，但仍保留数据

状态转换：
pending（待审核） → active（已上架）
pending（待审核） → rejected（已拒绝）
active（已上架） → rented（已出租）
rented（已出租） → active（已上架）
active（已上架） → inactive（已下架）
```

### 订单业务规则

#### 订单创建流程

```
1. 买家浏览账号，点击"查看详情"
2. 确认账号信息，点击"立即租赁"
3. 进入支付确认页面
4. 显示：
   - 账号信息（名称、等级、段位、哈夫币等）
   - 租期
   - 租金
   - 押金
   - 总价
5. 买家确认信息，点击"立即支付"
6. 调用支付API（Web端：微信JSAPI；小程序端：小程序支付）
7. 支付成功后，创建订单
8. 冻结买家押金到frozenBalance
9. 订单状态：paid（已支付）
10. 创建聊天群组（订单群）
11. 卖家在聊天中发送账号信息
12. 买家验证账号后，点击"确认验收"
13. 订单状态：active（进行中）
14. 开始计算租期
15. 24小时后，自动提醒买家归还
```

#### 订单时间轴

```
创建订单（createdAt）
  ↓
支付成功（paidAt）
  ↓
买家确认验收（startTime） - 租期开始
  ↓
进行中（active）
  ├─ 租期过半：发送提醒
  ├─ 租期结束前1小时：发送归还提醒
  └─ 租期结束（endTime）
      ↓
买家点击归还（verificationRequestTime）
  ↓
待验收（pending_verification） - 48小时内必须验收
  ↓
卖家验收
  ├─ 通过：订单完成
  └─ 拒绝：争议处理
      ↓
订单完成（completedAt）
  ↓
自动分账（settledAt）
```

#### 验收规则

```
买家验收（接收账号）：
- 支付成功后，卖家在聊天中发送账号信息
- 买家验证账号是否可用
- 买家点击"确认验收"
- 开始计算租期

卖家验收（归还账号）：
- 买家点击"归还账号"
- 订单状态变为pending_verification
- 卖家有48小时验收时间
- 卖家验收结果：
  - 通过：订单完成，自动分账
  - 拒绝：进入争议流程，管理员介入

争议处理：
- 买家和卖家无法达成一致
- 管理员介入审核
- 根据实际情况判定：
  - 买家责任：扣除押金，订单完成
  - 卖家责任：全额退款，订单取消
  - 双方协商：部分退款
```

#### 退款规则

```
退款场景：
1. 支付后30分钟内，买家取消订单
   - 全额退款（包括押金）
   - 订单状态：cancelled

2. 租期内，账号无法使用
   - 卖家配合解决：延长租期
   - 卖家无法解决：协商退款
   - 争议：管理员介入

3. 违规账号
   - 管理员取消订单
   - 全额退款
   - 账号下架

退款计算：
- 未开始租期：全额退款
- 已开始租期：按比例退款
  - 已使用时长 < 10%：退款90%
  - 已使用时长 < 30%：退款70%
  - 已使用时长 < 50%：退款50%
  - 已使用时长 >= 50%：不退款
```

#### 分账规则

```
分账时机：
- 订单状态变为completed
- 卖家验收通过

分账计算：
- 平台佣金 = 租金 × 佣金比例（默认5%）
- 卖家收入 = 租金 - 平台佣金
- 买家押金退还 = 押金（解冻）

示例：
订单总价：90元
- 租金：40元
- 押金：50元

分账：
- 平台佣金：40 × 5% = 2元
- 卖家收入：40 - 2 = 38元（转入availableBalance）
- 买家押金：50元（从frozenBalance解冻到availableBalance）
```

## 💳 支付流程

### 支付方式

```
Web端：
- 微信JSAPI支付
- 扫码支付（待开发）

小程序端：
- 微信小程序支付
```

### 支付流程

```
Web端支付流程：
1. 买家点击"立即支付"
2. 后端创建订单
3. 后端调用微信统一下单API
4. 返回支付参数（prepayId等）
5. 前端调用微信JSAPI
6. 用户扫码支付
7. 支付成功后，微信回调通知后端
8. 后端验证签名
9. 更新订单状态为paid
10. 冻结买家押金
11. 创建聊天群组
12. 前端轮询订单状态
13. 支付完成，跳转到订单详情

小程序端支付流程：
1. 买家点击"立即支付"
2. 后端创建订单
3. 后端调用微信统一下单API
4. 返回小程序支付参数（timeStamp, nonceStr, package, signType, paySign）
5. 前端调用wx.requestPayment
6. 用户支付
7. 支付成功后，微信回调通知后端
8. 后端验证签名
9. 更新订单状态为paid
10. 冻结买家押金
11. 创建聊天群组
12. 前端跳转到订单详情
```

### 支付回调处理

```
微信支付回调：
1. 接收回调通知
2. 验证签名
3. 检查订单是否存在
4. 检查订单是否已支付
5. 更新订单状态为paid
6. 记录支付信息
7. 冻结买家押金
8. 创建聊天群组
9. 返回成功响应

回调重试：
- 如果处理失败，微信会重试通知
- 最多重试15次
- 需要做好幂等性处理
```

```
小程序端提现：

5. 输入提现金额
   - 最低：10元
   - 最高：不超过availableBalance
6. 选择提现方式（微信零钱）
7. 确认提现
8. 扣除availableBalance
9. 记录交易记录
10. 调用微信商家转账API
11. 转账成功后，更新状态
12. 返回成功

提现规则：
- 仅实名认证用户可以提现
- 最低提现金额：10元
- 提现手续费：0（平台承担）
- 提现到账时间：即时
```

#### 余额冻结与解冻

```
冻结（下单时）：
- 买家下单支付成功
- 冻结押金到frozenBalance
- availableBalance不变
- 记录交易记录

解冻（订单完成时）：
- 订单完成
- 从frozenBalance转移到availableBalance
- 记录交易记录

解冻（订单取消时）：
- 订单取消
- 从frozenBalance转移到availableBalance
- 记录交易记录
```

#### 余额变动记录

```
每次余额变动都会记录：
- 用户ID
- 订单ID（如果是订单相关）
- 交易类型
- 变动金额
- 变动前余额
- 变动后余额
- 描述
- 时间

用户可以查看所有交易记录
```
