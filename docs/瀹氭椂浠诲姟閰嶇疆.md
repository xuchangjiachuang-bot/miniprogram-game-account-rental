## 配置定时任务

### 方案1：使用外部 Cron 服务（推荐）

#### 使用 EasyCron

1. 注册账号：[https://www.easycron.com](https://www.easycron.com)
2. 创建新的 Cron Job：
   - **URL**: `https://hfb.yugioh.top/api/orders/check-expired`
   - **执行频率**: `*/5 * * * *`（每 5 分钟执行一次）
   - **HTTP 方法**: `GET`

#### 使用 Cron-job.org

1. 访问：[https://cron-job.org](https://cron-job.org)
2. 登录后点击"Create cronjob"
3. 配置：
   - **Title**: 检查到期订单
   - **URL**: `https://hfb.yugioh.top/api/orders/check-expired`
   - **Execution times**: `*/5 * * * *`（每 5 分钟）
   - **HTTP Method**: `GET`

---

### 方案2：使用服务器 Cron（如果有服务器访问权限）

登录服务器后编辑 crontab：

```bash
crontab -e
```

添加以下内容：

```bash
# 每 5 分钟检查一次到期订单
*/5 * * * * curl -s https://hfb.yugioh.top/api/orders/check-expired
```

保存并退出。

---

### 方案3：使用 GitHub Actions（开源项目）

如果项目托管在 GitHub，可以使用 GitHub Actions：

创建文件 `.github/workflows/check-expired-orders.yml`：

```yaml
name: 检查到期订单

on:
  schedule:
    # 每 5 分钟执行一次（UTC 时间）
    - cron: '*/5 * * * *'
  workflow_dispatch:  # 允许手动触发

jobs:
  check-expired-orders:
    runs-on: ubuntu-latest
    steps:
      - name: 检查到期订单
        run: |
          curl -X GET https://hfb.yugioh.top/api/orders/check-expired
```

## 业务逻辑说明

### 1. 检查到期的 active 订单

- **订单状态**: `active` → `pending_verification`
- **设置验收截止时间**: 48 小时后
- **通知**: 需要单独实现通知逻辑

### 2. 检查超时的 pending_verification 订单

- **订单状态**: `pending_verification` → `completed`
- **执行分账**: 调用 `executeAutoSplit()`
- **退还押金**: 自动处理

## 监控建议

1. **定期检查日志**:
   ```bash
   tail -f /app/work/logs/bypass/app.log
   ```

2. **监控 API 响应时间**:
   - 正常响应时间应 < 2 秒
   - 如果响应时间过长，可能需要优化数据库查询

3. **监控失败次数**:
   - 如果持续失败，检查数据库连接
   - 检查是否有订单数据异常

---

## 注意事项

1. **执行频率**:
   - 建议：每 5 分钟
   - 最小：每 1 分钟
   - 最大：每 30 分钟

2. **时区问题**:
   - 确保定时任务使用服务器时间或 UTC 时间
   - 建议统一使用 UTC 时间

3. **错误处理**:
   - API 已实现错误处理
   - 不会因为单个订单失败而中断整个流程
   - 失败的订单会记录在日志中

4. **并发控制**:
   - API 是幂等的，可以多次执行
   - 不会重复处理同一个订单

## 故障排查

### 问题：定时任务没有执行

1. 检查定时任务配置是否正确
2. 检查服务器时间是否正确
3. 检查 URL 是否可访问

### 问题：订单没有被处理

1. 检查订单的 `endTime` 和 `verificationDeadline` 字段
2. 检查订单状态是否正确
3. 查看日志中的错误信息

### 问题：分账失败

1. 检查分账服务配置
2. 检查账户余额是否充足
3. 查看详细的错误日志
