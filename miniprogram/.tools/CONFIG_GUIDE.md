## 🔧 配置真实后端API

如果您已经部署了后端服务，按照以下步骤配置：

### 步骤1: 修改API配置

打开文件：`miniprogram/utils/config.js`

找到以下配置：

```javascript
const config = {
  // API 基础地址
  baseUrl: 'https://your-domain.com/api',

  // 使用Mock数据（测试用）
  useMockData: true,
```

**修改为您的真实API地址**：

```javascript
const config = {
  // API 基础地址 - 替换为您的后端API地址
  baseUrl: 'https://your-actual-domain.com/api',

  // 关闭Mock数据模式
  useMockData: false,
```

### 步骤2: 配置微信服务器域名

登录微信公众平台：https://mp.weixin.qq.com/

1. 进入：开发 → 开发管理 → 开发设置
2. 找到"服务器域名"
3. 在"request合法域名"中添加您的API域名：
   ```
   https://your-actual-domain.com
   ```
4. 保存后等待10-15分钟生效

### 步骤3: 重新编译

在微信开发者工具中：
1. 保存所有修改的文件（Ctrl+S）
2. 点击"编译"按钮
3. 等待编译完成

### 步骤4: 测试

编译完成后：
1. 点击"预览"按钮
2. 扫描二维码在手机上测试
3. 检查是否可以正常加载真实数据

## ❓ 常见问题

### 1. 修改配置后还是空白？

**可能原因**：
- API地址配置错误
- 服务器域名未在微信后台配置
- 后端服务未启动
- 网络连接问题

**解决方法**：
1. 检查 `config.js` 中的 `baseUrl` 是否正确
2. 确认已在微信后台配置服务器域名
3. 确认后端服务正常运行
4. 使用浏览器直接访问API地址测试

### 2. 如何查看错误信息？

在微信开发者工具中：
1. 点击底部的"调试器"标签
2. 查看"Console"控制台
3. 查看是否有红色错误信息
4. 查看"Network"网络请求，检查API请求状态

### 3. 首页数据加载失败？

**检查步骤**：
1. 确认 `useMockData` 设置为 `false`
2. 确认 `baseUrl` 配置正确
3. 确认服务器域名已在微信后台配置
4. 检查后端API是否正常响应

### 4. 临时使用Mock数据？

**保持 Mock 模式**：
- 将 `useMockData` 设置为 `true`
- 不需要配置后端API
- 可以完整展示所有功能界面

**用途**：
- 演示小程序UI效果
- 测试页面布局和交互
- 向客户展示原型
