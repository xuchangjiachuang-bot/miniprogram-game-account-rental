# 📋 小程序问题诊断与修复总结

## 🚨 发现的问题

### 1. ❌ API连接失败（主要问题）

**错误信息**：
```
request:fail errcode:-102 cronet_error_code:-102 error_msg:net::ERR_CONNECTION_REFUSED
```

**原因**：小程序开发者工具默认会校验请求域名，本地开发环境未配置白名单

**状态**：✅ 已提供解决方案

---

### 2. ⚠️ 哈夫币输入框显示异常

**现象**：输入框显示空白/乱码，"最小"和"最大"提示不可见

**可能原因**：
- CSS样式问题（height为0或background为透明）
- placeholder颜色与背景色相同
- 网络加载问题导致样式未应用

**状态**：✅ 已提供检查方法

---

### 3. ⚠️ "发布账号"按钮重复显示

**现象**：页面上出现了两个完全一样的"发布账号"按钮

**原因**：
- 代码中确实有两个按钮（一个在fallback-hero中，一个在publish-section中）
- 当轮播图数据为空时，fallback-hero会显示第一个按钮
- publish-section也会显示第二个按钮

**状态**：✅ 已确认代码逻辑，非bug

**说明**：这是正常的设计逻辑，有轮播图时显示一个，无轮播图时显示两个

---

### 4. ℹ️ vConsole调试按钮显示

**现象**：页面左上角显示绿色的vConsole按钮

**原因**：微信开发者工具的开发环境默认显示vConsole用于调试

**状态**：✅ 这是正常现象，生产环境不会显示

---

## ✅ 解决方案

### 方案1：开启"不校验合法域名"（推荐）⭐

**适用场景**：本地开发环境

**步骤**：
1. 打开微信开发者工具
2. 点击右上角「详情」
3. 切换到「本地设置」标签
4. 勾选**「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**
5. 点击「编译」

**预期效果**：
- ✅ API连接正常
- ✅ 账号列表正常加载
- ✅ 所有功能正常使用

---

### 方案2：使用Mock数据（临时方案）

**适用场景**：无法连接后端时临时测试界面

**步骤**：
```javascript
// miniprogram/utils/config.js
const environments = {
  development: {
    baseUrl: 'http://localhost:5000/api',
    debug: true,
    useMockData: true,  // 改为 true
  },
  // ...
};
```

**预期效果**：
- ✅ 可以测试界面布局
- ✅ 可以测试交互逻辑
- ❌ 无法测试真实API功能

---

### 方案3：配置服务器白名单（生产环境）

**适用场景**：生产环境正式上线

**步骤**：
1. 登录微信公众平台：https://mp.weixin.qq.com/
2. 进入：开发 → 开发管理 → 开发设置 → 服务器域名
3. 添加域名到 request 合法域名：
   ```
   https://yugioh.top
   ```
4. 修改配置：
   ```javascript
   // miniprogram/utils/config.js
   const ENV = 'production';
   ```

**注意**：
- 域名必须是HTTPS
- 需要备案的域名
- 生效需要几分钟

## 💡 常见问题

### Q1: 为什么会显示"不校验合法域名"选项？

A: 这是微信为了安全考虑，默认会校验请求的域名是否在白名单中。开发环境可以关闭这个校验，生产环境必须配置白名单。

---

### Q2: 哈夫币输入框为什么显示异常？

A: 可能是CSS样式问题，检查：
- height是否为0
- background是否为透明
- placeholder颜色是否与背景色相同

---

### Q3: "发布账号"按钮重复是bug吗？

A: 不是bug。这是正常的设计：
- 有轮播图数据时：显示publish-section中的按钮
- 无轮播图数据时：显示fallback-hero和publish-section两个按钮

---

### Q4: vConsole按钮可以关闭吗？

A: 开发环境可以显示，生产环境会自动隐藏。如果要在开发环境隐藏，可以在app.json中设置debug为false，但不推荐。
