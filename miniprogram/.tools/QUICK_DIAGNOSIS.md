## 问题概览

### 1. 🔴 API连接失败（主要问题）
**错误**: `errcode:-102 net::ERR_CONNECTION_REFUSED`

**根本原因**:
- 微信小程序开发者工具默认会校验请求域名
- 本地开发环境`localhost:5000`未配置白名单
- 请求被拦截导致连接失败

**✅ 解决方案（推荐）**:
1. 打开微信开发者工具
2. 点击右上角「详情」
3. 切换到「本地设置」标签
4. ✅ 勾选**「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**
5. 点击「编译」
6. 点击「清缓存」→「全部清除」

**预期效果**:
- ✅ API连接正常
- ✅ 账号列表正常加载
- ✅ 所有功能正常使用

### 2. 🟡 哈夫币输入框显示异常
**现象**: 输入框显示空白/乱码

**可能原因**:
- CSS样式问题（height为0或background为透明）
- placeholder颜色与背景色相同

**✅ 检查方法**:
1. 打开vConsole（点击页面左上角绿色按钮）
2. 检查Console是否有错误
3. 检查Elements标签，查看input元素的样式
4. 确认height、line-height、background等属性

**修复建议**:

  font-size: 28rpx !important;
  color: #1f2937 !important;
}

.havoc-coin-input::placeholder {
  color: #9ca3af !important;

### 3. 🟢 "发布账号"按钮重复显示
**现象**: 页面上出现了两个完全一样的"发布账号"按钮

**根本原因**:
- 代码中确实设计了两个按钮（非bug）
- 当轮播图数据为空时，fallback-hero显示第一个按钮
- publish-section显示第二个按钮
- 这是正常的设计逻辑，用于不同的展示场景

**✅ 状态**: 正常现象，无需修复

**说明**:
- 有轮播图数据时：只显示publish-section中的按钮
- 无轮播图数据时：显示fallback-hero和publish-section两个按钮
- 点击任何一个按钮都会触发相同的提交逻辑

---

### 4. 🟢 vConsole调试按钮显示
**现象**: 页面左上角显示绿色的vConsole按钮

## 💡 关键提示

### 关于"不校验合法域名"

**为什么需要勾选？**
- 微信小程序为了安全，默认会校验请求的域名
- 开发环境的`localhost:5000`未在白名单中
- 勾选后可以绕过校验，方便开发调试

**生产环境怎么办？**
- 需要在微信公众平台配置服务器域名白名单
- 添加域名到 request 合法域名
- 使用HTTPS协议
- 域名需要备案

---

### 关于vConsole按钮

**可以关闭吗？**
- 开发环境：可以显示，方便调试
- 生产环境：自动隐藏，用户看不到
- 不建议在开发环境关闭

**vConsole有什么用？**
- 查看Console日志
- 查看Network请求
- 查看Elements元素
- 查看Storage存储
- 查看System信息
