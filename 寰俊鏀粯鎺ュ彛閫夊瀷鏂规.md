# 微信支付接口选型方案

## 一、项目概述

三角洲行动哈夫币出租平台，主要功能：
- 卖家上架账号，买家租赁账号
- 平台抽佣和自动分账
- 卖家提现

## 二、支付场景分析

### 1. 买家支付（租金+押金）
- **金额**：租金（订单金额）+ 押金（账号价值的50%）
- **场景**：用户在移动端下单支付
- **频率**：高频交易

### 2. 卖家提现
- **金额**：可提现余额
- **场景**：卖家申请提现到微信零钱
- **频率**：中频操作

### 3. 平台分账
- **金额**：订单金额的5%（平台佣金）
- **场景**：订单完成后自动分账
- **频率**：与订单同步

## 三、推荐方案

### 方案：混合支付模式

#### 1. 收款端（买家支付）
采用 **H5支付 + JSAPI支付** 组合

**H5支付**（移动浏览器）
- 适用：用户在手机浏览器访问
- 流程：下单 → 调用H5支付 → 跳转微信支付页 → 支付成功 → 回调

**JSAPI支付**（微信内置浏览器）
- 适用：用户在微信内访问
- 流程：下单 → 调用JSAPI支付 → 微信内弹窗支付 → 支付成功 → 回调

**优点**：
- 覆盖所有移动端用户
- 用户体验好
- 开发难度适中

**缺点**：
- H5支付需要跳转，体验略差

#### 2. 分账端（订单分账）
采用 **微信支付分账接口**

**流程**：
- 订单完成 → 调用分账接口 → 平台分账到卖家账户

**优点**：
- 支持多人分账
- 自动化程度高
- 资金安全

**注意事项**：
- 需要开通分账权限
- 分账比例需要在商户平台配置
- 分账时间为订单完成后（建议24小时内）

#### 3. 提现端（卖家提现）
采用 **商家转账到零钱**

**流程**：
- 卖家申请提现 → 管理员审核 → 调用转账接口 → 资金到账

**优点**：
- 直接转账到微信零钱
- 用户体验好

**注意事项**：
- 需要开通转账权限
- 需要用户授权（获取openid）
- 需要风控措施（防止恶意提现）

## 四、技术实现方案

### 1. 支付流程

```
用户下单 → 创建订单 → 调用微信支付 → 支付成功 → 更新订单状态 → 24小时后自动分账
```

### 2. 提现流程

```
卖家申请提现 → 人工审核 → 调用转账接口 → 提现成功 → 更新余额
```

### 3. 分账流程

```
订单完成 → 创建分账请求 → 平台分账给卖家 → 更新分账记录
```

### 4. 技术栈

- **SDK**：官方微信支付SDK（npm install wechatpay-node-v3）
- **签名方式**：微信支付v3 API（SHA256-RSA）
- **回调处理**：Webhook接收支付结果
- **对账**：每日对账单下载

## 五、成本分析

### 费率
- **支付费率**：0.6%（普通商户）
- **分账费率**：免费（平台内部分账）
- **转账费率**：0.1%（转账到零钱）

### 开通成本
- **企业微信支付商户号**：免费（需要企业认证）
- **分账权限**：免费（需申请）
- **转账权限**：免费（需申请）

## 六、风险控制

### 1. 支付风控
- 订单金额限制（单笔不超过5000元）
- 同一用户下单频率限制（每小时不超过5次）
- 异常订单监控

### 2. 提现风控
- 实名认证（必须完成实名认证才能提现）
- 单笔提现限额（不超过10000元）
- 每日提现限额（不超过50000元）
- 提现审核（人工审核大额提现）

### 3. 分账风控
- 分账金额验证
- 分账账号验证
- 分账失败重试机制

## 七、开发计划

### 第一阶段：基础支付（1-2周）
1. 集成H5支付和JSAPI支付
2. 实现支付回调处理
3. 完善订单状态管理

### 第二阶段：分账功能（1周）
1. 集成分账接口
2. 实现自动分账逻辑
3. 添加分账记录查询

### 第三阶段：提现功能（1周）
1. 集成转账接口
2. 实现提现申请和审核
3. 添加提现记录查询

### 第四阶段：对账和监控（1周）
1. 实现每日对账功能
2. 添加异常交易监控
3. 完善日志和报表

## 八、注意事项

### 1. 商户号配置
- 需要企业微信支付商户号
- 需要配置商户API密钥
- 需要上传商户证书（用于退款和转账）

### 2. 权限申请
- 分账权限：需要在商户平台申请
- 转账权限：需要在商户平台申请
- 审核时间：1-3个工作日

### 3. 测试环境
- 微信支付提供沙箱环境
- 建议先在沙箱环境测试
- 测试通过后再切换到正式环境

### 4. 安全性
- API密钥不要泄露
- 使用HTTPS通信
- 验证回调签名
- 订单金额验证

## 九、总结

推荐采用 **H5支付 + JSAPI支付 + 分账接口 + 转换接口** 的组合方案：

1. **收款端**：H5支付 + JSAPI支付（覆盖所有移动端用户）
2. **分账端**：微信支付分账接口（自动化分账）
3. **提现端**：商家转账到零钱（快速提现）

该方案的优势：
- 用户体验好
- 自动化程度高
- 资金安全
- 成本可控

预计开发周期：4-5周
预计开发成本：中等（主要是集成和测试）
预计上线时间：商户号审核通过后1个月内
