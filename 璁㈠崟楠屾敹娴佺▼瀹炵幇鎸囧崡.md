# 订单验收流程实现指南

## 业务背景

### 问题说明
之前的流程存在严重缺陷：
- 买家归还账号后立即完成订单并分账
- 卖家无法检查账号状态
- 如果账号异常（物品丢失、被封号），无法追回资金

### 正确流程
```
1. 买家点击"归还账号"
   - 订单状态：active → pending_verification（待验收）
   - 通知卖家验收（48小时内）

2. 卖家验收（三种选择）
   a) 确认正常
      - 订单状态：pending_verification → completed
      - 执行分账
      - 退还买家押金
   
   b) 发现异常
      - 订单状态：pending_verification → disputed（纠纷）
      - 卖家上传证据
      - 平台审核
      - 根据审核结果扣除押金或正常分账
   
   c) 超时未验收（48小时后）
      - 订单自动完成
      - 执行分账
      - 退还买家押金
```

## 已完成的功能

### 1. 数据库表设计 ✅

#### orders表新增字段
```sql
ALTER TABLE orders
ADD COLUMN verification_request_time TIMESTAMP,
ADD COLUMN verification_deadline TIMESTAMP,
ADD COLUMN verification_result VARCHAR(20) DEFAULT 'pending',
ADD COLUMN verification_remark TEXT,
ADD COLUMN dispute_evidence JSONB;
```

**字段说明**：
- `verification_request_time`：买家归还账号的时间（验收请求时间）
- `verification_deadline`：验收截止时间（48小时后）
- `verification_result`：验收结果（pending-待验收，passed-通过，rejected-拒绝）
- `verification_remark`：验收备注
- `dispute_evidence`：纠纷证据（JSON格式）

### 2. 新增订单状态 ✅

**pending_verification**（待验收）
- 触发条件：买家归还账号或租期到期
- 持续时间：最长48小时
- 状态流转：
  - → completed（卖家确认正常）
  - → disputed（卖家发现异常）
  - → completed（超时自动）

### 3. API接口 ✅

#### 1. 归还账号API
**接口**：`POST /api/orders/[id]/complete`

**参数**：
```json
{
  "action": "complete"
}
```

**响应**：
```json
{
  "success": true,
  "message": "账号归还成功，等待卖家验收（48小时内完成验收）",
  "data": {
    "orderId": "xxx",
    "orderNo": "xxx",
    "status": "pending_verification",
    "verificationDeadline": "2025-01-16T09:00:00Z",
    "note": "卖家验收通过后，订单将自动完成并分账；若发现异常可发起纠纷"
  }
}
```

**业务逻辑**：
1. 检查订单状态（必须为active）
2. 更新订单状态为`pending_verification`
3. 设置验收截止时间（当前时间 + 48小时）
4. 记录验收请求时间
5. 通知卖家验收

#### 2. 卖家验收API
**接口**：`POST /api/orders/[id]/verify`

**参数**（验收通过）：
```json
{
  "action": "pass",
  "remark": "验收通过，账号状态正常"
}
```

**参数**（验收拒绝）：
```json
{
  "action": "reject",
  "remark": "验收失败，账号物品丢失",
  "evidence": {
    "images": ["url1", "url2"],
    "description": "账号内原本有10个皮肤，现在只剩5个"
  }
}
```

**响应**（验收通过）：
```json
{
  "success": true,
  "message": "验收通过，订单已完成并分账",
  "data": {
    "orderId": "xxx",
    "orderNo": "xxx",
    "status": "completed",
    "platformCommission": 5.00,
    "sellerIncome": 95.00,
    "buyerRefund": 50.00
  }
}
```

**响应**（验收拒绝）：
```json
{
  "success": true,
  "message": "验收已拒绝，纠纷已创建，平台将介入审核",
  "data": {
    "orderId": "xxx",
    "orderNo": "xxx",
    "status": "disputed",
    "note": "平台将在24小时内审核纠纷，审核完成后将根据结果处理资金"
  }
}
```

**业务逻辑**：
1. 检查订单状态（必须为`pending_verification`）
2. 验收通过：
   - 更新订单状态为`completed`
   - 更新验收结果为`passed`
   - 执行分账（平台佣金 + 卖家收入）
   - 退还买家押金
3. 验收拒绝：
   - 更新订单状态为`disputed`
   - 更新验收结果为`rejected`
   - 记录纠纷证据
   - 创建纠纷记录
   - 等待平台审核

#### 3. 到期订单检查API（已更新）
**接口**：`GET /api/orders/check-expired`

**新增功能**：
1. 检查到期的`active`订单：
   - 订单状态：`active` → `pending_verification`
   - 设置验收截止时间（48小时后）
   - 通知卖家验收

2. 检查超时的`pending_verification`订单：
   - 订单状态：`pending_verification` → `completed`
   - 验收结果：`passed`（自动通过）
   - 执行分账
   - 退还买家押金

**响应**：
```json
{
  "success": true,
  "message": "检查完成，到期订单：2个，超时自动完成：1个，失败：0个",
  "expiredActiveCount": 2,
  "autoVerifiedCount": 1,
  "failedCount": 0
}
```

### 详细步骤

#### 1. 买家归还账号
```typescript
// 前端调用
fetch(`/api/orders/${orderId}`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ action: 'complete' })
});

// 后端处理
1. 检查订单状态（必须为active）
2. 更新订单状态为pending_verification
3. 设置验收截止时间（48小时后）
4. 返回成功响应
```

#### 2. 卖家验收通过
```typescript
// 前端调用
fetch(`/api/orders/${orderId}/verify`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    action: 'pass',
    remark: '验收通过，账号状态正常'
  })
});

// 后端处理
1. 检查订单状态（必须为pending_verification）
2. 更新订单状态为completed
3. 调用分账服务：
   - 计算平台佣金（5%）
   - 计算卖家收入（95%）
   - 更新卖家余额
   - 退还买家押金
4. 返回成功响应
```

#### 3. 卖家验收拒绝
```typescript
// 前端调用
fetch(`/api/orders/${orderId}/verify`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    action: 'reject',
    remark: '验收失败，账号物品丢失',
    evidence: {
      images: ["url1", "url2"],
      description: "账号内原本有10个皮肤，现在只剩5个"
    }
  })
});

// 后端处理
1. 检查订单状态（必须为pending_verification）
2. 更新订单状态为disputed
3. 记录纠纷证据
4. 创建纠纷记录
5. 通知平台审核
6. 返回成功响应
```

#### 4. 超时自动验收
```typescript
// 定时任务调用（每5分钟）
fetch('/api/orders/check-expired');

// 后端处理
1. 查询所有超时的pending_verification订单
2. 更新订单状态为completed
3. 调用分账服务
4. 返回处理结果
```

## 风险控制

### 1. 验收时效
- **验收期限**：48小时
- **超时处理**：自动验收通过
- **通知机制**：
  - 买家归还后立即通知卖家
  - 验收截止前1小时提醒卖家

### 2. 证据留存
- 卖家验收拒绝必须上传证据
- 证据包括：截图、视频、详细描述
- 证据存储在数据库（JSON格式）

### 3. 纠纷处理
- 平台审核时间：24小时
- 审核结果：
  - 卖家胜诉：扣除买家押金（部分或全部），赔偿卖家
  - 买家胜诉：订单完成，执行分账
  - 双方协商：按照协商结果处理

### 4. 资金冻结
- 待验收期间，资金处于冻结状态
- 验收通过后，立即解冻并分账
- 纠纷期间，资金继续冻结，直到审核完成

## 注意事项

### 1. 数据一致性
- 订单状态更新必须原子操作
- 分账必须在验收通过后执行
- 押金退还必须在订单完成时执行

### 2. 异常处理
- 验收超时：自动完成
- 分账失败：记录日志，人工处理
- 纠纷审核：人工介入

### 3. 用户体验
- 清晰的提示文案
- 实时进度显示
- 及时的通知提醒
