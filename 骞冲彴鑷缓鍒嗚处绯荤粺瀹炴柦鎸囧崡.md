# 平台自建分账系统实施指南

## 业务流程

### 完整订单流程

```
1. 用户下单支付
   ↓
2. 卖家确认（上传二维码）
   ↓
3. 买家扫码开始租赁（订单状态：active）
   ↓
4. 租赁期间（可能持续1天、7天、30天等）
   ↓
5. 订单完成（两种方式）
   - 方式一：租期到期自动完成（系统定时任务检查）
   - 方式二：用户手动归还账号（用户主动完成）
   ↓
6. 执行分账（立即执行）
   - 计算平台佣金（5%）
   - 卖家收入到账（95%）
   - 退还买家押金
   ↓
7. 卖家申请提现
   ↓
8. 管理员审核（小额自动审核）
   ↓
9. 转账到零钱（微信支付）
```

**关键点**：
- ✅ 分账在**订单完成时立即执行**，不是支付后24小时
- ✅ 订单完成时机：租期到期 OR 用户手动归还
- ✅ 定时任务每5分钟检查一次到期订单
- ✅ 提现审核：≤1000元自动通过，>1000元人工审核

## 已完成的功能

### 1. 数据表设计 ✅
- **orders表**：添加了 `platform_fee`、`is_settled`、`settled_at` 字段
- **split_records表**：记录分账信息（已存在）
- **withdrawals表**：记录提现信息（已存在）
- **user_balances表**：管理用户余额（已存在）

### 2. 核心服务 ✅

#### 平台分账服务 (`src/lib/platform-split-service.ts`)
- `executeAutoSplit(orderId)`：执行订单分账
- `getSplitStatus(orderId)`：查询分账状态

**分账流程**：
1. 订单完成后自动调用
2. 计算平台佣金（默认5%）
3. 计算卖家收入（95%）
4. 更新卖家可用余额
5. 退还买家押金
6. 记录分账记录

#### 用户余额服务 (`src/lib/user-balance-service.ts`)
- `getUserBalance(userId)`：获取用户余额
- `addAvailableBalance(userId, amount)`：增加可用余额
- `freezeBalance(userId, amount)`：冻结余额
- `unfreezeBalance(userId, amount)`：解冻余额
- `requestWithdrawal(userId, amount, accountInfo)`：申请提现
- `reviewWithdrawal(withdrawalId, approved, reviewerId, remark)`：审核提现

#### 微信支付服务 (`src/lib/wechat-pay-service.ts`)
- `createJsapiPayment(params)`：创建JSAPI支付订单
- `createH5Payment(params)`：创建H5支付订单
- `verifyPaymentNotify(...)`：验证支付回调
- `transferToBalance(params)`：转账到零钱
- `queryTransferResult(outBatchNo)`：查询转账结果

### 3. API接口 ✅

#### 订单管理API
- `GET /api/orders/check-expired`：检查并自动完成到期的订单（定时任务调用）
- `GET /api/orders/[id]`：获取订单详情
- `POST /api/orders/[id]/complete`：手动完成订单并分账

#### 用户端API
- `POST /api/withdrawals`：申请提现
- `GET /api/withdrawals`：获取提现记录

#### 管理端API
- `POST /api/admin/withdrawals/[id]/review`：审核提现
- `GET /api/admin/withdrawals/[id]`：获取提现详情

### 1. 部署定时任务（重要）

为了自动完成到期的订单，需要部署定时任务调用 `/api/orders/check-expired` 接口。

**方案一：使用 crontab（推荐）**

```bash
# 编辑 crontab
crontab -e

# 添加定时任务（每5分钟检查一次到期订单）
*/5 * * * * curl -s http://localhost:5000/api/orders/check-expired >> /var/log/order-check.log 2>&1
```

**方案二：使用 Vercel Cron Jobs**

创建 `vercel.json` 文件：

```json
{
  "crons": [
    {
      "path": "/api/orders/check-expired",
      "schedule": "*/5 * * * *"
    }
  ]
}
```

**方案三：使用 GitHub Actions**

创建 `.github/workflows/check-expired-orders.yml`：

```yaml
name: Check Expired Orders

on:
  schedule:
    - cron: '*/5 * * * *'

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - name: Check expired orders
        run: curl -X GET https://your-domain.com/api/orders/check-expired
```

## 使用说明

### 订单完成和分账流程

**订单完成时机**：
1. **租期到期自动完成**：系统定时检查到期的订单，自动完成并分账
2. **用户手动归还**：用户提前归还账号，手动完成订单并分账

**自动完成到期的订单**：
```typescript
// 定时任务调用此API（例如每5分钟执行一次）
// GET /api/orders/check-expired
const response = await fetch('/api/orders/check-expired');
const result = await response.json();

console.log(`检查完成，成功 ${result.completedCount} 个订单`);
```

**用户手动完成订单**：
```typescript
// POST /api/orders/[id]/complete
const response = await fetch(`/api/orders/${orderId}`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    action: 'complete'
  })
});

const result = await response.json();

if (result.success) {
  console.log('订单完成，分账成功', {
    platformCommission: result.data.platformCommission,
    sellerIncome: result.data.sellerIncome,
    buyerRefund: result.data.buyerRefund
  });
}
```

### 订单分账流程

订单完成后，系统自动执行分账：

```typescript
import { executeAutoSplit } from '@/lib/platform-split-service';

// 订单完成后自动执行分账
const result = await executeAutoSplit(orderId);

if (result.success) {
  console.log('分账成功', {
    platformCommission: result.platformCommission,
    sellerIncome: result.sellerIncome,
    buyerRefund: result.buyerRefund
  });
}
```

### 提现申请流程

```typescript
import { requestWithdrawal } from '@/lib/user-balance-service';

// 用户申请提现
const result = await requestWithdrawal(
  userId,
  100,  // 提现100元
  {
    openid: 'user_openid'  // 微信openid
  }
);

if (result.success) {
  console.log('提现申请成功', {
    amount: result.amount,
    fee: result.fee,
    actualAmount: result.actualAmount
  });
}
```

### 提现审核流程

```typescript
import { reviewWithdrawal } from '@/lib/user-balance-service';

// 管理员审核提现
const result = await reviewWithdrawal(
  withdrawalId,
  true,  // 审核通过
  adminId,
  '审核通过'
);

if (result.success) {
  console.log('提现审核通过');

  // 审核通过后，调用微信转账接口
  const transferResult = await transferToBalance({
    outBatchNo: withdrawalId,
    openid: 'user_openid',
    amount: result.actualAmount,
    description: '提现'
  });

  if (transferResult.success) {
    console.log('转账成功', transferResult.batchId);
  }
}
```

## 成本分析

### 单笔订单（租金100元 + 押金50元）
- 支付手续费：150 × 0.6% = **0.9元**
- 平台佣金：100 × 5% = **5元**
- 转账手续费：95 × 0.1% = **0.095元**
- **平台净收入**：5 - 0.9 = **4.1元**
- **卖家净收入**：94.905元

### 风险控制

#### 1. 提现审核
- **小额提现**（≤1000元）：自动审核通过
- **大额提现**（>1000元）：人工审核

#### 2. 余额管理
- 分开核算平台抽佣和用户资金
- 建立资金池备付金
- 每日对账

#### 3. 异常监控
- 异常订单监控
- 异常提现监控
- 余额异常监控

## 注意事项

### 1. 微信支付配置
- 必须使用已备案的域名
- 必须配置HTTPS证书
- 需要配置支付授权目录
- 需要配置回调地址

### 2. 安全性
- 不要泄露API密钥和证书
- 使用HTTPS通信
- 验证回调签名
- 订单金额验证

### 3. 合规性
- 初期（日均交易<100笔）：使用自建分账系统
- 中期（日均交易100-1000笔）：申请微信支付服务商
- 长期（日均交易>1000笔）：使用银行存管账户
